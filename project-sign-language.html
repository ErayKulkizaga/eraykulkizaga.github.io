SignTurk
Turkish Sign Language Platform

An award-winning, real-time Turkish Sign Language recognition platform built with FastAPI, a browser-based interface, HF RTMW/RTMPose whole-body landmark extraction, and a 226-class Keras ensemble. SignTurk combines live translation, prediction history, administration, a sign dictionary, and 3D avatar experiences in one product.

Live Recognition

The browser streams webcam frames to the FastAPI backend through WebSocket. The backend extracts whole-body and hand landmarks, builds skeleton/motion feature streams, runs the SignTurk ensemble, and returns top predictions with confidence values.

  • Webcam-to-WebSocket live inference flow
  • HF RTMW / RTMPose whole-body landmark extraction
  • Top predictions, confidence values, and live diagnostics
  • SQLite fallback when Supabase is unavailable

Dictionary & Avatar

The application also includes a dictionary-style landmark endpoint and a 3D avatar screen. Sign animation data is served from local landmark JSON and smoothed before playback for clearer motion.

  • Local model/landmarks.json dictionary data
  • Smoothed landmark playback endpoint
  • Three.js avatar assets stored with Git LFS

Product Experience

Runtime Pipeline

1

Frame Capture

The browser interface captures webcam frames and sends them to /api/predict/live. The live path is designed to keep only a small number of frames in flight so the UI stays responsive.

2

Landmark Extraction

HF RTMW/RTMPose extracts whole-body landmarks. The active layout is pose33 + left_hand21 + right_hand21, represented as 75 x 4 values for x, y, z, and confidence.

3

Feature Builder

The runtime converts landmarks into joint, bone, joint-motion, bone-motion, and extra streams. Orientation controls are available for camera mirroring and left/right hand debugging.

4

Ensemble Prediction

The ensemble combines skeleton_v2, skeleton_v4_seed, skeleton_v6_seed, and hand_stream Keras models to produce a 226-class Turkish Sign Language prediction.
